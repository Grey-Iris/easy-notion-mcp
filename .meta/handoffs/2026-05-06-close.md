## Review notes

Fresh code review initially found 3 issues:

1. `profile check` used broad `search("", "pages")`, which could enumerate a whole workspace.
2. CLI `content append` skipped MCP's `file://` upload preprocessing.
3. Hidden `--trust-content` opt-out suppressed the untrusted-content notice.

All three were fixed before commit:

- `profile check` now uses `getMe` and optionally `getPage(root_page_id)`; otherwise it returns `read_probe: { ok: true, skipped: "no_root_page_id" }`.
- `content append` now runs `processFileUploads(..., "stdio")` before `markdownToBlocks`.
- `--trust-content` is rejected; `page read` always includes the untrusted content notice.

Final fresh review found no material issues. Residual risk: CLI tests are mock-based, so live SDK error-message redaction is not exercised; no direct token-printing path was found.
