## Deep child write slice

Commit: `8c0ddea fix(notion): defer nested block children`

Files changed:

- `src/notion-client.ts`
- `tests/notion-client-block-chunking.test.ts`

Behavior:

- Child-bearing optional containers (`bulleted_list_item`, `numbered_list_item`, `toggle`, toggleable headings) are created without children, then descendants are appended recursively to returned block IDs.
- Direct child appends still chunk at 100 and preserve order.
- `table` writes seed with the first row and append additional rows later.
- `column_list` / `column` writes use required seed children while deferring deeper unsafe content.
- A column whose first child is a `table` or `column_list` now gets a safe empty paragraph seed; the original child subtree is appended later to avoid over-deep payloads.
- `appendBlocksAfter` preserves the top-level `after` cursor while nested children are deferred.
- `createPage` rollback still handles deferred nested append failures by trashing the partial page and rethrowing the original error.

- `5ae0ce`: Always avoid sending nested Notion write payloads deeper than two child levels; when a container's required seed child would itself require nested children, use a safe placeholder seed and defer the original child subtree into later append calls.
